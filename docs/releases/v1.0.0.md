# 🎉 v1.0.0 — Initial Release

**VA HCPCS Fee Schedule Manager**
*VISN 22 / WSNC IMPACT Team*

> A standalone Windows desktop application for VA staff to manage, view, filter, and export CMS DMEPOS HCPCS fee schedule data.

---

## 📥 Download

| File | Description |
|---|---|
| **`HCPCSFeeApp-Setup.zip`** | ✅ Recommended — contains the app, installer, and icon |
| `HCPCSFeeApp.exe` | Standalone executable (for advanced users) |

### Installation (no admin rights required)

1. Download **`HCPCSFeeApp-Setup.zip`** from the Assets section below.
2. Extract the ZIP to any temporary location.
3. Double-click **`Install.bat`** to run the installer.
4. The app will be installed to `Documents\HCPCSFeeApp\` and a desktop shortcut will be created.
5. Double-click the **VA HCPCS Fee Schedule Manager** shortcut on your desktop to launch.

> **Note:** No administrator rights are needed. Everything installs to your user profile.

## ✨ Features

### CMS Data Sync
- **Auto-download** CMS DMEPOS fee schedules for any combination of the 50 US states + DC
- **Supported years:** 2024 through the current calendar year
- **Multi-layer URL discovery** — RSS feed → HTML scraping → pattern tracker → hardcoded templates
- **24-hour URL cache** to avoid redundant CMS requests
- **Pattern tracker** — learns from successful downloads and adapts when CMS changes URL conventions
- **Quarterly replace** — each sync replaces prior data for the same year/state, keeping results current without duplicates
- **Smart file detection** — selects the primary DMEPOS CSV from CMS ZIP files while excluding Rural ZIP, Former CBA, PEN, and other auxiliary datasets

### Data Management
- **Import CSV** — load existing VISN-format CSVs or manually downloaded CMS files with auto-delimiter detection
- **Filter** by state, year, HCPCS code, HCPCS group, and description keyword
- **ZIP code lookup** — enter a 5-digit ZIP to automatically determine rural (R) vs. non-rural (NR) allowable amounts
- **State management** — select which states to track via `Settings → Manage States`
- **Year management** — select which fiscal years to track via `Settings → Manage Years`
- **Import log** — view a history of all data imports with timestamps and record counts

### Export
- **CSV** — standard comma-separated export
- **Excel (.xlsx)** — formatted workbook with headers and styling
- **PDF** — print-ready report with VISN branding

### Developer Tools
- **SQL Publisher** — run direct SQL queries against the local database
- **Databricks / ODBC publishing** — push data to external endpoints (requires `pyodbc` and appropriate drivers)

### Application
- **SQLite database** — all data stored locally, no server or network connection required after sync
- **First-run setup wizard** — guided state and year selection on first launch
- **Desktop shortcut creation** — available from `Settings → Create Desktop Shortcut`
- **Configurable database path** — move the database folder via `Settings → Change Database Path`
- **In-app update notifications** — the app checks GitHub for new releases on startup and shows an update bar when a newer version is available
- **One-click self-update** — click "Update Now" to download and apply updates without leaving the app
- **Splash screen** — branded WSNC VISN 22 map loading screen with progress bar during startup

## 🔄 Updating

When a new version is available, the app will display a notification bar at the top of the window:

> 🔔 **Update available!** Version X.Y.Z is ready. **[Update Now]**

Click **"Update Now"** to download and apply the update automatically. The app will restart with the new version. No need to re-download the ZIP or re-run the installer.

## 🐛 Known Limitations

- The `.exe` is not code-signed. If downloading the standalone `.exe` (not the ZIP), Windows SmartScreen may show an "Unknown publisher" warning. Choose **More info → Run anyway** if permitted. The ZIP-based install avoids this for most users.
- The SQL Publisher / Databricks features require `pyodbc` and an appropriate ODBC driver installed on the machine, plus network credentials for the target endpoint.
- If CMS changes their website structure significantly, the auto-download may fail. The app will show a clear error with a link to manually download from CMS, which can then be imported via **File → Import CSV**.
